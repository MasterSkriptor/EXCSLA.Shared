# Copilot/AI Onboarding Documentation Plan

## Context
- Audience: Copilot, AI agents, automation scripts, advanced users
- Goal: Enable automated refactoring and onboarding for EXCSLA.Shared, especially MediatR removal and Dispatcher adoption
- Requirements: Explicit, actionable instructions; example prompts; migration checklists
- References: EXCSLA repo, MediatR docs, Dispatcher usage examples

## Plan Steps
1. Document how to automate MediatR removal and Dispatcher adoption
2. Provide example Copilot prompts for refactoring and migration
3. Create a checklist for Copilot-driven migration
4. Make instructions explicit and actionable for AI agents
5. Update or create a CHANGELOG.md for documentation changes
6. Specify review/approval workflow for documentation changes
7. Verify all code snippets build and run as shown
8. Ensure documentation accessibility (headings, alt text, tables)
9. Collect feedback from users and Copilot after major updates
10. Archive or link to old documentation versions as needed
11. (Optional) Automate checks for formatting, broken links, and code snippet validity

12. Ensure all code compiles and passes tests before pushing to GitHub
