# Phase 1, Task 3: Create EXCSLA.Shared.Core.Exceptions

## Context
This task is part of Phase 1 of the migration. Refer to `00_Migration_Introduction.txt` for the overall goal and guiding principles.

## Task
Create the `EXCSLA.Shared.Core.Exceptions` project to centralize custom domain exceptions.

## Actions
1.  Move all custom domain exceptions into `src/Core/EXCSLA.Shared.Core.Exceptions/`.
2.  Create a new `EXCSLA.Shared.Core.Exceptions.csproj` file in this directory.
3.  Update the solution file to include this new project.

## Acceptance Criteria
- This project must only contain exception classes.
- This project may depend on `EXCSLA.Shared.Core.Abstractions`.
- The solution must build without errors after the changes.
